# Protocol V2 功能映射

本文说明当前软件功能如何对应 CTDP 与 RSIP 方法论。Protocol 是行为协议工具，用于记录、执行和复盘用户自己设定的行为规则；它不是医疗诊断、心理治疗或健康干预软件。

## CTDP：链式时延协议

### 主链

主链对应 CTDP 的 Main Chain。V2 Beta 中，每条主链不再只是“名称 + 时长”，而是一份完整协议：

- 链名称
- 神圣座位触发动作
- 持续时间
- 完成条件
- 内嵌辅助链配置

启动主链后，系统创建一条活跃 `focus_sessions` 记录。该记录会保存当次启动时的触发动作和完成条件快照，避免之后编辑主链时改写旧历史。

### 神圣座位

神圣座位对应 CTDP 的“标志性触发动作”。它不是独立数据表、独立页面状态或新的会话类型，而是主链正式任务的产品化表达。

- 主链完成：写入 `focus_sessions.completed`，主链当前长度加一。
- 主链失败并裁定违规：写入 `focus_sessions.failed_reset`，主链当前长度清零。
- 主链失败并判例化：写入 `focus_sessions.failed_precedent`，新增一条主链判例，主链当前长度不清零。

### 辅助链

辅助链对应“线性时延原理”。它把“现在开始”的高阻力动作转化为“稍后必须开始”的低阻力承诺。

V2 Beta 中，辅助链是主链详情页里的内嵌配置，不再有单独的预约启动页。底层仍复用 `reservation_sessions`，以保持历史事件和旧数据兼容。

- 辅助链履约：写入 `reservation_sessions.fulfilled`，并进入主链神圣座位。
- 辅助链未履约：写入 `reservation_sessions.failed_reset`，主链长度不受影响。
- 辅助链判例化：写入 `reservation_sessions.failed_precedent`，新增一条辅助链判例。

每条主链当前只有一个辅助链配置。V2 Beta 仍不维护独立辅助链长度。

### 判例法

判例法对应“下必为例”原则。软件不替用户判断某个行为是否违约，而是要求用户做出正式裁决。

- 判定违规：承认本次行为破坏协议，并执行主链清零或辅助链失败记录。
- 判例化：承认该行为以后在同类情形中允许，并保存到判例库。

链详情页会把主链判例和辅助链判例合并展示为“协议边界”，并用标签区分来源。

### 失败调试

主链失败和辅助链未履约裁决后，系统会记录轻量调试信息：

- 触发动作太重
- 完成条件过高
- 时间太长
- 环境不适合
- 规则不清
- 状态不足
- 其他

调试信息只用于复盘，不会自动修改主链或辅助链配置。

## RSIP：递归稳态迭代协议

### 定式节点

定式节点对应 RSIP 中的“国策 / 定式”。每个定式应是明确、低阻力、可执行的小规则。

当前字段包括：

- 标题
- 执行说明
- 父节点
- 点亮状态
- 创建、点亮、熄灭时间

### 定式格

定式格对应 RSIP 的递归回滚结构。用户可以创建根定式，也可以把新定式挂到已有定式下面。

- 根节点代表可以独立存活的基础定式。
- 子节点代表依赖父节点稳态而更容易存在的后续定式。
- 页面使用缩进树展示定式之间的依赖关系。

### 点亮、熄灭与回滚

定式点亮表示它当前处于生效状态。定式熄灭表示用户裁定该定式当前未能维持。

当一个 active 父节点熄灭时，系统会递归熄灭其所有 active 子孙节点，并写入 `rollback_child_deactivated` 事件。这对应 RSIP 的回滚机制：父节点坍塌时，依赖它的后续节点同步退回。

### 事件日志

`formula_events` 记录 RSIP 的关键事件：

- `created`：定式加入树
- `activated`：定式点亮
- `deactivated`：定式熄灭
- `rollback_child_deactivated`：因父节点熄灭触发递归回滚

Dashboard 会展示 RSIP 摘要，History 会把 CTDP 与 RSIP 事件合并成统一协议时间线。

## V2 Beta 边界

本版本实现的是 CTDP 主链 / 辅助链配置升级与轻量失败调试，不包含：

- 独立辅助链长度
- 二次预约信号
- 通知、窗口置顶、系统锁定或退出拦截
- RSIP 定式向导
- 水密隔舱
- 国策组
- AI 建议
- 医疗评估、诊断或治疗建议

这些内容可以在后续阶段继续扩展，但不应破坏当前主链、辅助链与判例闭环。
